# 分布式锁的实现方式

## 前言

> 在日常工作中，经常会碰到多线程并发的情况，如何保证数据在同一时间只有一个线程访问？一般的解决方案就是枷锁。例如 java 中的` synchronized关键字 ` `Reentrantlock可重入锁`等等的一些  JUC 类，但是这些锁只能在同一个 JVM 中才会生效，对于分布式的服务中，并不能保证线程的同步执行。对于分布式场景，可以使用分布式锁。



## 分布式锁的实现方式

分布式锁一般有三种实现方式

* 数据库乐观锁
* 基于Redis实现的分布式锁
* 基于 Zookeeper实现的分布式锁
